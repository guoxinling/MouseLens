## 1. 产品概述

`MouseLens` 是一款面向 `macOS` 的轻量录屏产品，核心价值不是“录屏”，而是“把一段普通的屏幕操作自动处理成更像演示视频的成片”。

它要解决的不是传统剪辑软件的问题，而是这条链路的问题：

`录屏 -> 手动剪辑 -> 放大重点 -> 做镜头运动 -> 包装背景 -> 导出`

`MouseLens` 希望把这条链路收敛成：

`录屏 -> 自动生成镜头 -> 微调 -> 导出`

## 2. 对标方向

参考 `Screen Studio` 的不是全部功能，而是以下产品思想：

- 默认效果优先于复杂自由度
- 录完就有“像做过后期”的观感
- UI 极简，参数少但有效
- 录制、编辑、导出是一体化工作流
- 适合产品演示、教程、社媒视频，而不是重剪辑

`MouseLens` 的差异化方向：

- 更轻
- 更快上手
- 更本地优先
- 更适合中文用户
- 后续可走更低价或一次性买断路线

## 3. 产品目标

### 3.1 核心目标

让用户在 `3 分钟内` 完成一条可发布的操作演示视频。

### 3.2 商业目标

- 做成一个 `单人可持续维护` 的桌面工具
- 先完成 `可卖的 MVP`
- 再逐步补齐“像正式产品”的体验，而不是一开始做成庞大复杂软件

### 3.3 成功指标

- 新用户首次安装后，`10 分钟内` 跑通第一次录制和导出
- 默认参数下，`70%+` 的录制无需进入复杂编辑即可导出
- 导出成功率 > `95%`
- 权限链路成功率 > `90%`

## 4. 目标用户

### 4.1 核心用户

- 独立开发者
- SaaS 创始人
- 产品经理
- 教程作者
- 运营 / 增长 / 市场同学

### 4.2 高频场景

- 演示一个产品新功能
- 录一个 onboarding 教程
- 做一段社媒产品介绍短视频
- 给客户或团队展示操作流程

### 4.3 不优先用户

- 专业视频剪辑师
- 影视后期团队
- 需要多轨时间线和复杂特效的人

## 5. 用户问题

当前用户使用录屏工具时常见问题：

- 录出来只是“原始屏幕”，没有重点
- 要去剪辑软件里手动做缩放和镜头移动
- 录屏工具和剪辑工具割裂
- 出片很慢，做一次演示成本太高
- 为了简单录个 demo，却被迫学习复杂软件

## 6. 产品原则

- 默认值优先：不依赖用户精细调参
- 极简优先：不要做成 `Premiere`
- 一体化优先：录制、编辑、导出尽量不跳出
- 本地优先：先把本地流程打磨好，再考虑云
- 可卖优先：先做最常用的 `80%`，不做炫技功能

## 7. 核心价值主张

### 对市场

“比通用剪辑软件快很多，比纯录屏软件好看很多。”

## 9. 核心工作流

### 9.1 标准流程

1. 用户打开 `MouseLens`
2. 选择录制目标和基础参数
3. 点击 `Start Recording`
4. 倒计时后自动开始录制
5. 用户完成操作后停止录制
6. 系统自动生成镜头方案
7. 用户在编辑页微调
8. 点击导出
9. 得到可播放的成片

### 9.2 菜单栏流程

1. 用户通过菜单栏图标快速开始
2. 录制中不需要回主窗口
3. 用快捷键停止录制
4. 自动回到编辑页

## 13. 核心交互要求

### 13.1 首页

- 用户必须在首页一眼看懂怎么开始录
- 权限异常必须能被明确解释
- 默认配置应可直接使用

### 13.2 编辑页

- 不能像传统剪辑软件那样复杂
- 控件数量要克制
- 调参数后应尽量快速看到反馈

### 13.3 菜单栏

- 必须支持“不开主窗口也能开始一条录制”
- 状态要一目了然：空闲、倒计时、录制中

## 14. 非目标

以下内容在 `v1.0` 前不应成为重点：

- 多轨时间线
- 复杂剪辑
- 调色系统
- 音频混音台
- 云协作
- 重型 AI 套件

## 15. 商业模式建议

### 15.1 早期建议

- 免费试用
- 基础导出可体验
- 高级导出或无水印付费

### 15.2 定价建议

不建议一开始走极低月费。

更稳妥的方向：

- 方案 A：一次性买断 `¥39 - ¥99`
- 方案 B：年费 `¥69 - ¥149`
- 方案 C：免费版 + Pro 年费

### 15.3 竞争策略

不要只打“更便宜”。

应该同时打：

- 更简单
- 更轻
- 更快上手
- 更适合中文用户
- 本地优先

## 16. 风险与挑战

### 16.1 技术风险

- macOS 权限和签名复杂
- 录制与导出兼容性问题多
- 不同显示器 / 缩放 / 多屏场景复杂
- QuickTime / 系统播放器兼容性要求高

### 16.2 产品风险

- 功能越加越多，容易变成“小 Premiere”
- 如果默认效果不好，用户不会愿意调参
- 如果首条录制成功率不高，留存会很差

### 16.3 商业风险

- 赛道不大
- 易被大厂或成熟工具覆盖
- 单靠低价未必有足够利润

## 17. 当前建议的开发优先级

按优先顺序：

1. 继续提升成片默认效果
2. 完善菜单栏和录制中体验
3. 完善设置页和首启引导
4. 做窗口录制和异常恢复
5. 做摄像头气泡

## 19. 一句话总结

`MouseLens` 不该做成另一个剪辑软件，而应该做成一个“默认就很好看”的演示录屏工具。

`用户录完一条视频，第一次就觉得：这已经可以发了。`
